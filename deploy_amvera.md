# Инструкция по деплою на Amvera

---

## Шаг 1. Создать проект

1. Зайти на [amvera.ru](https://amvera.ru) → **«Создать проект»**
2. Тип: **Python**
3. Имя: произвольное (например `frog-converter`)

---

## Шаг 2. Подключить GitHub-репозиторий

В разделе **«Репозиторий»** выбрать вкладку **GitHub**:

1. **Получить токен GitHub (PAT):**
   - Перейти на [github.com/settings/tokens](https://github.com/settings/tokens)
   - Нажать **«Generate new token (classic)»**
   - Отметить галочку **`repo`**
   - Нажать «Generate token», скопировать (формат `ghp_...` — это нормально)

2. **Вставить токен** в поле «Токен» → нажать **«Применить»**

3. В поле **«Удалённый репозиторий»** выбрать нужный репозиторий

4. В поле **«Ветка»** выбрать `main`

5. Нажать **«Подключить»**

---

## Шаг 3. Убедиться в корректности amvera.yml

Файл `amvera.yml` должен находиться в **корне репозитория** и содержать:

```yaml
meta:
  environment: python
  toolVersion: 3.11
  toolchain:
    name: pip
run:
  containerPort: 8000
  command: uvicorn main:app --host 0.0.0.0 --port 8000
```

### Частые ошибки конфигурации

| Ошибка | Причина | Решение |
|---|---|---|
| `must contain section 'meta.toolchain'` | Отсутствует блок `meta.toolchain` | Добавить `toolchain: name: pip` |
| `Invalid variable found: installDependencies` | Поля `build.installDependencies` не существует | Удалить весь блок `build` |
| Приложение не открывается по URL | `--host 127.0.0.1` вместо `0.0.0.0` | Использовать `--host 0.0.0.0` |

---

## Шаг 4. Задать переменные окружения

В разделе **«Переменные»**:

| Имя переменной | Значение |
|---|---|
| `QWEN_API_KEY` | Ваш ключ из [ModelStudio](https://modelstudio.console.alibabacloud.com/) → API Keys |

> ⚠️ Не добавляйте `.env` файл в репозиторий — ключ должен задаваться только через интерфейс Amvera.

---

## Шаг 5. Запустить деплой

После подключения репозитория и задания переменных Amvera запускает сборку автоматически. Если нет — нажать кнопку **«Развернуть»** (иконка в верхней панели).

Процесс сборки:
1. Amvera клонирует репозиторий
2. Выполняет `pip install -r requirements.txt`
3. Запускает `uvicorn main:app --host 0.0.0.0 --port 8000`

---

## Шаг 6. Получить URL

После успешного деплоя в разделе **«Домены»** появится адрес:
```
https://<project-name>.amvera.io
```
Также можно подключить собственный домен.

---

## Обновление приложения

При каждом `git push` в ветку `main` Amvera **автоматически пересобирает** и переразворачивает приложение.

```bash
git add .
git commit -m "обновление"
git push
```

---

## Проверка логов при проблемах

- **«Логи» → «Лог сборки»** — ошибки при `pip install` или в `amvera.yml`
- **«Логи» → «Лог приложения»** — ошибки в рантайме (видны когда статус «Приложение запускается» или «Запущено»)
- **«Логи» → «Лог системы»** — системные события
