# Функциональное задание: DOCX Summarizer

## 1. Суть проекта

Веб-приложение для пакетной обработки файлов DOCX:
1. Пользователь загружает до 20 файлов DOCX
2. Приложение конвертирует их в TXT
3. Каждый TXT отправляется в DeepSeek API для суммаризации
4. Результаты сохраняются в CSV-таблицу
5. Пользователь скачивает TXT-файлы (архивом) и/или CSV-таблицу

---

## 2. Пользовательский сценарий (шаг за шагом)

1. Открыть сайт в браузере
2. Нажать «Выбрать файлы DOCX» → выбрать до 20 файлов
3. Нажать **«Конвертировать»** → файлы конвертируются в TXT на сервере
4. Появляется кнопка **«Скачать TXT»** → скачивается ZIP-архив с TXT-файлами
5. Нажать **«Отправить на суммаризацию»** → файлы по одному уходят в DeepSeek, прогресс отображается в лог-блоке
6. После завершения появляется кнопка **«Скачать таблицу»** → скачивается CSV
7. При ошибке отдельного файла — в логе отображается сообщение об ошибке, остальные файлы продолжают обрабатываться

## 4. Серверная логика (Backend)

### 4.1 Конвертация DOCX → TXT
- Принимает массив DOCX-файлов
- Конвертирует каждый через библиотеку `python-docx`
- Возвращает TXT-файлы в виде ZIP-архива
- Сохраняет TXT временно в памяти сервера (в рамках сессии запроса)

### 4.2 Суммаризация через DeepSeek API
- Принимает массив TXT-файлов (или текстов)
- Отправляет каждый файл отдельным запросом в DeepSeek
- **Промпт для DeepSeek** (системная инструкция):
  ```
  Сделай суммаризацию текста на русском языке. 
  Объём — не более 1000 символов. 
  Структурируй по логическим разделам документа, используя короткие подзаголовки. 
  Отвечай только суммаризацией, без вводных фраз.
  ```
- При ошибке одного файла — записывает `"Ошибка обработки"` в CSV и продолжает
- Возвращает CSV-файл

### 4.3 Формат CSV
- Кодировка: UTF-8 с BOM (для корректного открытия в Excel)
- Разделитель: `;` (стандарт для русской локали Excel)
- Колонки:
  - `Имя файла` — оригинальное имя без расширения
  - `Суммаризация` — текст от DeepSeek

## 5. Архитектура

```
Браузер (HTML + JS)
       ↕  HTTP запросы (fetch)
FastAPI сервер (Python)
       ↕
DeepSeek API (внешний сервис)
```

**Почему такая архитектура:**
- Вся тяжёлая работа (конвертация, запросы к API) на сервере — браузер не перегружается
- API-ключ DeepSeek хранится только в переменных окружения сервера, в браузер не попадает
- FastAPI — простой и быстрый Python-фреймворк, хорошо поддерживается на Amvera

## 7. Описание каждого файла

### `main.py`
Содержит три HTTP-эндпоинта:

| Метод | Путь | Что делает |
|---|---|---|
| `GET` | `/` | Отдаёт `index.html` |
| `POST` | `/convert` | Принимает DOCX-файлы, возвращает ZIP с TXT |
| `POST` | `/summarize` | Принимает TXT-файлы, возвращает CSV |

Переменная окружения: `DEEPSEEK_API_KEY`

### `requirements.txt`
```
fastapi
uvicorn
python-docx
httpx
python-multipart
```

### `amvera.yml`
```yaml
meta:
  environment: python
  toolVersion: "3.11"
run:
  persistenceMount: /data
  containerPort: 8000
build:
  installDependencies: true
  buildDirectory: .
```

### `static/index.html`
Одностраничный интерфейс. Использует `fetch()` для общения с сервером.  
Все стили и скрипт — внутри одного файла (как в исходной версии).

---

## 8. Ограничения и допущения

- Максимум **20 файлов** за один сеанс
- Максимальный размер одного файла — **10 МБ** (ограничение FastAPI по умолчанию)
- Файлы **не сохраняются** на сервере между запросами — только в памяти во время обработки
- Язык интерфейса — **русский**
- Суммаризация работает с документами на **любом языке**, но ответ всегда на русском

| Переменная | Значение |

| `DEEPSEEK_API_KEY` | Ваш ключ из личного кабинета DeepSeek |

Добавляется в разделе **«Переменные окружения»** в настройках приложения на Amvera.

## 10. Порядок разработки для агента в IDE

1. Создать `main.py` с тремя эндпоинтами (конвертация, суммаризация, отдача статики)
2. Создать `static/index.html` на основе прилагаемого исходного файла, добавив новые кнопки и логику
3. Создать `requirements.txt`
4. Создать `amvera.yml`
5. Протестировать локально: `uvicorn main:app --reload`
6. Задеплоить на Amvera через Git или загрузку архива
